# Design Concept

## 프로젝트 디자인 컨셉

이 프로젝트의 현재 UI 컨셉은 **미니멀 코퍼레이트 리퀴드 글래스**입니다.

핵심 방향은 다음 두 가지를 동시에 만족하는 것입니다.

- 리퀴드 글래스 특유의 투명감, 얇은 반사광, 부드러운 깊이감
- 삼성 DS / S.LSI 리더 과정에 어울리는 차분하고 보고서 같은 기업형 톤

즉, 화려한 쇼케이스형 글래스 UI가 아니라, **업무용 실습 도구에 맞는 절제된 글래스 인터페이스**를 목표로 합니다.

## 핵심 키워드

- Bright
- Calm
- Executive
- Structured
- Light Glass
- Soft Depth
- Low Noise

## 비주얼 원칙

### 1. 정보가 먼저 보이고, 효과는 뒤에 머문다

시각 효과는 존재하지만 전면에 나서지 않습니다.

- 제목, 설명, 입력 영역, 결과 영역의 구분이 먼저 읽혀야 합니다.
- 글래스 효과는 정보 구조를 보조하는 수준으로 유지합니다.
- 장식성보다 가독성과 위계가 우선입니다.

### 2. 밝은 배경 위의 얇은 유리층

전체 화면은 밝은 블루-그레이 계열 배경 위에 반투명 유리 패널이 겹쳐진 구조입니다.

- 배경은 매우 연한 그라디언트와 약한 색 번짐만 사용합니다.
- 패널은 흰색 기반의 반투명 표면으로 구성합니다.
- 경계선은 진한 선보다 얇고 밝은 유리 라인으로 표현합니다.

### 3. 기업형 절제

리퀴드 글래스 효과를 사용하더라도 과한 입체감이나 컬러 번짐은 피합니다.

- 강한 네온 컬러 사용 지양
- 과도한 그림자, 과한 하이라이트 지양
- 장식적 애니메이션 최소화
- 안정적인 정렬과 여백 유지

## 컬러 방향

### 배경

- 매우 밝은 블루-화이트 계열
- 차가운 회색보다는 약간의 블루 톤을 가진 깨끗한 업무용 배경

### 텍스트

- 본문 텍스트는 진한 네이비/차콜 계열
- 보조 텍스트는 블루-그레이 계열
- 대비는 충분히 확보하되, 검정에 가까운 무거운 느낌은 피함

### 포인트 컬러

- 주 포인트: 블루
- 보조 상태색: 그린, 앰버, 레드
- 포인트 컬러는 버튼, 진행 상태, 상태 배지처럼 기능적 위치에서만 사용

## 재질감 원칙

현재 UI의 글래스 표현은 다음 요소의 조합으로 구성됩니다.

- 반투명 흰색 표면
- 얇은 밝은 테두리
- 상단 방향의 약한 하이라이트
- 낮은 강도의 소프트 섀도
- 제한적인 blur

중요한 점은, 이 재질감이 “젖은 유리”처럼 화려하게 보이는 것이 아니라, **정제된 프레젠테이션 슬라이드 위의 유리 패널**처럼 보이도록 유지하는 것입니다.

## 레이아웃 원칙

### 헤더

- 첫 화면에서 가장 강한 정보 앵커 역할
- 제목, 과정 성격, 한 줄 설명만 간결하게 제시
- 과도한 장식보다 브랜드 톤과 안정감을 전달

### 메인 패널

- 각 탭은 하나의 작업 단위로 인식되어야 함
- 하나의 큰 패널 안에서 입력과 결과 흐름이 자연스럽게 이어져야 함
- 카드가 난립하는 느낌보다, 큰 컨테이너 안의 섹션 분할 구조를 선호

### 하단 네비게이션

- 모바일 중심 사용성을 고려한 고정형 구조
- 떠 있는 유리 바처럼 보이되 과도하게 장식적이지 않게 유지
- 활성 탭은 살짝 강조하되 전체 바와 분리되지 않도록 정제된 강조 사용

## 타이포그래피 원칙

- 명확하고 안정적인 한글 가독성 우선
- 굵은 제목, 정돈된 본문, 짧은 라벨 구조 유지
- 헤드라인은 강하지만 공격적이지 않게
- 라벨과 보조 텍스트는 정보 구획을 돕는 수준으로만 사용

현재 방향은 “실습용 서비스 UI”보다는 “리더 교육용 워크스페이스”에 가깝습니다.

## 컴포넌트별 해석

### 패널

- 가장 큰 글래스 표면
- 전체 정보 구조를 담는 메인 캔버스

### Step 카드

- 설명을 보조하는 얇은 서브 패널
- 왼쪽 컬러 포인트로 중요도만 전달
- 본문보다 앞서 튀지 않도록 절제

### 입력창 / 선택창

- 실무 도구처럼 명확하고 안정적으로 보여야 함
- 유리층 안에 있으나 실제 입력 컨트롤은 비교적 또렷해야 함
- 배경 장식보다 조작 가능성이 먼저 느껴져야 함

### 버튼

- 기본 버튼은 기능 중심
- 주 버튼만 확실한 블루 계열 강조
- 보조 버튼은 밝은 유리 버튼처럼 보이되 본문보다 튀지 않게 구성

### 결과 박스

- 입력 영역과 구분되지만 같은 디자인 시스템 안에 있어야 함
- 생성된 프롬프트나 로그는 실무 문서처럼 읽히는 것이 중요
- 코드/로그 영역은 본문보다 약간 차분하고 밀도 있게 구성

## 사용성 원칙

- 모바일에서 먼저 읽히고 눌리도록 설계
- 작은 화면에서도 계층 구조가 무너지지 않아야 함
- 터치 대상은 충분히 크고 명확해야 함
- 화려한 효과 때문에 텍스트 대비가 떨어지지 않아야 함

## 지양할 것

다음 방향은 현재 컨셉과 맞지 않습니다.

- 지나치게 미래지향적인 네온 UI
- 과한 3D 유리 반사
- 색이 많은 대시보드 스타일
- 카드가 여러 겹 중첩되는 복잡한 구성
- 장식용 애니메이션 남발
- 지나치게 마케팅 랜딩페이지 같은 카피 톤

## 앞으로 유지해야 할 기준

이 디자인을 확장하거나 수정할 때는 아래 기준을 유지합니다.

1. 새로운 요소를 추가해도 전체가 하나의 밝은 글래스 시스템처럼 보여야 한다.
2. 정보 구조가 먼저 읽히고, 시각 효과는 그 다음이어야 한다.
3. 블루 계열 포인트는 행동 유도와 상태 표현에만 제한적으로 사용한다.
4. 리더 교육용 도구답게 안정적이고 신뢰감 있는 인상을 유지한다.
5. “멋있다”보다 “정돈되어 있다”는 느낌이 먼저 들어야 한다.

## 한 줄 요약

**현재 디자인은 밝고 절제된 기업형 리퀴드 글래스 UI이며, 실습 도구의 가독성과 구조를 해치지 않는 범위 안에서만 유리 질감을 사용하는 것이 핵심이다.**
